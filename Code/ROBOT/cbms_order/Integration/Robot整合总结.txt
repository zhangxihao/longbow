1. 关键字没有注释，写出来是给人看的吗？
2. Page层应该只是封装元素，Action层是动作，有点混乱
3. 前期底层动作不统一，导致Robot整合的时候太多关键字相同而导致脚本失败。
4. 对于二次开发来说应该只需要关注Page与Flow，Action应该放在公共维护。自己可以单独在原Action上进行二次封装，但是尽量少，如果这种比较多，大家达成一致直接封装成公用的。（例如：有些人喜欢打开浏览器后最大化，这样的两个动作如果达成共识可以封装进一个动作里）
5. 对于能直接用goto的用goto，不要使用过多的点击操作。目前更多的是关注功能，而不是交互，如果页面排版布局修改了，那么你的脚本不是要改。
6. 页面跳转到另一个页面去操作按钮之前请加上一个 WaitUntil Element Is Visible，这样可以避免N多sleep在那里睡着没意义。(不稳定，没有sleep好用)
7. 元素定位最好放Page层维护，测试用例里面只维护测试数据。
8. 请不要在用例里面写“死”的东西，除非你是想把用例写死。（例如：图片位置、文件位置...，还有人竟然写死链接>_<...）



1. 账号这些的话最好放公共区域，能支持配置就好了。(暂时可以通过变量的方式解决)

3. 测试初始化脚本应该加进去，例如前期的创建用户，修改金额。执行完毕后的删除测试数据等。
4. 后期如果大家一起开发在提交到服务端时，最好log能详细一点。
